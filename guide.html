版本 P1 · 适用于 60 个双盲抽检案例

进入标注页面

标注目标：判断系统输出是否能由页面展示的公开指令、候选连接和规则卡合理支持。你不需要判断道路是否与现实完全一致，也不需要寻找隐藏真值。

1. 开始前

  1. 使用电脑端 Chrome、Edge 或 Firefox。
  2. 填写唯一标注者 ID，例如 reviewer_01，不同标注者不得共用 ID。
  3. 独立完成全部 60 个案例，不与另一位标注者讨论答案。
  4. 进度保存在当前浏览器；不要使用无痕模式或清理浏览器数据。
  5. 完成后导出 JSON 并发送给研究负责人，CSV 仅用于自行检查。

2. 页面四部分

区域含义关注点
公开指令自然语言拓扑要求。要求添加、重映射、保留、拒绝、不操作，还是暂缓？
公开候选可选的局部车道级连接。起终道路、车道、方向及结构条件是否匹配。
规则卡检索到的自然语言交通修复知识。是否与指令、操作和理由相关。
系统输出决策、注册操作、候选、规则和理由组成的操作契约。操作与候选是否共同符合公开证据。

3. 候选字段速查

字段含义与判断
same_junction起终道路是否在同一交叉口；false 通常不能直接连接。
geometry几何是否支持该转向；false 表示证据不足。
existing连接是否已存在；重复添加通常应为 NO_OP。
uturn是否掉头；无明确依据时通常应拒绝或暂缓。
protected是否处于信号控制等受保护上下文。
lane_available车道是否可用；false 时不应物化。
dirl 左转、r 右转、s 直行、t 掉头、? 不明确。

4. 六种注册操作

操作用途
ADD_LANE_CONNECTION唯一确定缺失连接时添加连接。
REMAP_LANE_CONNECTION方向合理，但车道映射需要替换。
NO_OP无需修改，例如连接已存在。
REJECT_UNSUPPORTED_UTURN拒绝缺少依据的掉头连接。
PRESERVE_SIGNAL_CONTROL修复时保留信号控制。
DEFER_FOR_CLARIFICATION信息不足以唯一决策，需要补充信息。

5. 七项标注

1）指令自然性（1–5）

只评价指令表达，不评价系统答案。1=严重不自然或难懂；3=基本可理解但生硬；5=自然且清楚。

2）公开信息能否支持唯一判断

  • yes：足以唯一确定操作和目标。
  • no：关键信息缺失。
  • ambiguous：存在多个同样合理的解释或候选。

3）注册操作是否合理

只判断操作类别；证据不足时暂缓通常合理。

4）候选绑定是否合理

检查候选 ID 是否与指令中的道路、车道和转向一致。不需要候选时选 not_applicable。

5）规则卡是否相关

规则应支持当前操作。只支持部分理由选 partially；无需规则时选 not_applicable。

6）是否疑似泄漏目标答案

正常候选属性、指令中的候选 ID 和系统选择不算泄漏。页面若出现“gold”“正确答案”“oracle”“private reference”等评价者专用信息，才选 yes。

7）解释是否支持最终决策

理由应引用正确事实、候选属性和规则，并能推导到最终操作。结论正确但理由错误时选 no 或 partially。

6. 推荐顺序

  1. 先读指令，独立判断操作类别。
  2. 检查候选道路、车道、方向和结构属性。
  3. 判断是否存在唯一候选；否则考虑暂缓、拒绝或不操作。
  4. 检查规则卡能否支持该结论。
  5. 最后查看系统输出，分别评价操作、候选、规则和解释。
重要边界：评价“公开证据下的决策合理性”，不是复原现实道路。不要查询地图、原始 SUMO 网络、论文代码或隐藏真值。

7. 常见例子

  • 只有一个未存在候选完全匹配明确指令：通常添加连接。
  • 连接已存在但指令要求添加：通常 NO_OP。
  • 多个候选都可能是目标但缺少车道信息：通常 DEFER_FOR_CLARIFICATION。
  • 掉头候选没有许可或明确指令：通常拒绝。
  • 信号交叉口没有相位修改证据：应保留信号控制。

  • 标注者 ID 唯一。
  • 已导出包含标注者 ID 的 JSON。
  • 没有与另一位标注者交换答案。
